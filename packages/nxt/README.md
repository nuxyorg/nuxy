# `nxt` — Nuxy Extension Tooling

> Like `vsce` for VS Code, but for **Nuxy** extensions.

## Installation

```bash
# From the Nuxy repo (local development)
pnpm add -g ./packages/nxt

# Once published to npm:
npm install -g @nuxy/nxt
```

## Commands

| Command       | Description                                  |
| ------------- | -------------------------------------------- |
| `nxt info`    | Show extension metadata from `manifest.json` |
| `nxt package` | Bundle + sign into a `.nuxyext` file         |
| `nxt install` | Copy `.nuxyext` to `~/.nuxy/extensions/`     |
| `nxt keys`    | Generate a new RSA developer key pair        |
| `nxt publish` | _(coming soon)_ Publish to the Nuxy registry |

## Workflow

```bash
# Inside any extension directory:
cd extensions/calendar

# 1. See what will be packaged
nxt info

# 2. Create the .nuxyext bundle
nxt package

# 3. Install locally into the running Nuxy shell
nxt install
```

### Options

```bash
# Custom keys file
nxt package --keys ~/my-keys.json

# Output to a specific directory
nxt package --out ./dist

# Skip signing (not recommended for distribution)
nxt package --no-sign
```

## How it works

`nxt package` reads your extension's `manifest.json`, zips all source files
(excluding `node_modules`, `.git`, and test files), signs the bundle with your
RSA developer key, and writes a `<extension-id>.nuxyext` file.

The key pair is auto-discovered in this order:

1. `--keys <path>` option
2. `.nxt-keys.json` in the current directory
3. `../../dist/developer-keys.json` (repo-level dev keys, auto-generated by `pnpm dev`)

`nxt install` copies the `.nuxyext` into `~/.nuxy/extensions/`, which the Nuxy
shell monitors and hot-reloads automatically.

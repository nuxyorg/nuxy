# Extension Packaging

The **`nxt`** CLI (`@nuxy/nxt`) bundles, signs, and installs Nuxy extensions. It lives in `packages/nxt` and is the `vsce`-equivalent for Nuxy.

## Installation

```bash
# From the Nuxy repo (local development)
pnpm add -g ./packages/nxt

# Once published to npm
npm install -g @nuxy/nxt
```

## Commands

All commands run from inside an extension directory (where `manifest.json` lives).

| Command       | Description                                  |
| ------------- | -------------------------------------------- |
| `nxt info`    | Show extension metadata from `manifest.json` |
| `nxt package` | Bundle backend + sign → `.nuxyext` file      |
| `nxt install` | Copy `.nuxyext` to `~/.nxy/extensions/`      |
| `nxt keys`    | Generate a new RSA developer key pair        |
| `nxt publish` | _(coming soon)_ Publish to the Nuxy registry |

## Workflow

```bash
cd extensions/clipboard

# 1. Inspect what will be packaged
nxt info

# 2. Create the .nuxyext bundle
nxt package

# 3. Install into the running Nuxy shell
nxt install
```

## nxt package

```bash
# Default — bundles backend, signs, writes to dist/
nxt package

# Custom output directory
nxt package --out ./releases

# Custom signing keys
nxt package --keys ~/my-keys.json

# Skip signing (not recommended for distribution)
nxt package --no-sign
```

The output file is `dist/<extension-id>-<version>.nuxyext`, for example:

```
dist/com.nuxy.clipboard-1.2.0.nuxyext
```

### Build steps

```
1. Read manifest.json
2. Bundle backend entry → _backend.bundle.mjs  (esbuild, ESM, node platform, deps inlined)
3. Load or generate RSA signing keys
4. Zip all source files (excluding node_modules, .git, tests, dist)
5. Sign → signature.json added to the ZIP
6. Write <ext-id>.nuxyext to --out directory
```

### Bundle contents

```
com.nuxy.clipboard-1.2.0.nuxyext (ZIP)
  manifest.json
  backend.ts               # original source
  _backend.bundle.mjs      # compiled, self-contained backend
  settings.json            # if present
  locales/
    en.json
  signature.json           # RSA signature + per-file SHA-256 integrity map
```

Files excluded from the bundle: `node_modules/`, `.git/`, `tests/`, `dist/`.

## Code signing

Extensions are signed with an RSA-2048 key pair. `nxt package` auto-discovers keys in this order:

1. `--keys <path>` option
2. `../../dist/developer-keys.json` (repo-level dev keys, auto-generated by `pnpm dev`)
3. `.nxt-keys.json` in the extension directory (auto-generated if missing)

The `signature.json` written into the ZIP contains the RSA signature, the public key, and a SHA-256 hash of every bundled file.

To generate a standalone key pair:

```bash
nxt keys
# writes .nxt-keys.json to the current directory

nxt keys --out ~/nuxy-dev-keys.json
```

To skip signing entirely (local testing only):

```bash
nxt package --no-sign
```

## nxt install

Copies the `.nuxyext` into `~/.nxy/extensions/` and removes any previously installed version of the same extension. Nuxy hot-reloads the extension automatically.

```bash
nxt install
# auto-detects dist/<ext-id>.nuxyext

nxt install --file ./dist/com.nuxy.clipboard.nuxyext
```

::: tip Symlink detection
If `~/.nxy/extensions/` is a symlink (created by the dev setup), `nxt install` replaces it with a real directory so the installed file is not silently written into the dev source tree.
:::

## External dependencies

If your extension uses third-party npm packages, declare them in the extension's own `package.json`:

```json
{
  "dependencies": {
    "fuse.js": "^7.0.0"
  }
}
```

`nxt package` resolves and inlines all dependencies into `_backend.bundle.mjs` via esbuild's bundle mode. No `node_modules` folder is shipped.

::: tip Keep bundles small
Third-party dependencies increase install size. Prefer packages with small footprints.
:::

## Versioning

The version in `manifest.json` controls the output filename. Bump it before packaging a new release — if the output file already exists, the packager exits with an error. Use `--force` to overwrite.

```bash
nxt package          # fails if dist/com.nuxy.clipboard-1.2.0.nuxyext already exists
nxt package --force  # overwrites
```

```json
{
  "id": "com.nuxy.clipboard",
  "version": "1.3.0"
}
```

## CI integration

```yaml
# .github/workflows/pack.yml
- name: Install nxt
  run: npm install -g ./packages/nxt

- name: Package extension
  working-directory: extensions/clipboard
  run: nxt package

- name: Upload release asset
  uses: actions/upload-release-asset@v1
  with:
    upload_url: ${{ steps.create_release.outputs.upload_url }}
    asset_path: extensions/clipboard/dist/com.nuxy.clipboard.nuxyext
    asset_name: com.nuxy.clipboard.nuxyext
    asset_content_type: application/zip
```

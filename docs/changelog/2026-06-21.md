# Changelog — 2026-06-21

## `angrysearch` — port to current extension architecture

- Ported the legacy `angrysearch` file-search extension (Voidtools Everything-style, SQLite FTS4-backed) from the old React + hooks architecture into a fresh `extensions/angrysearch/` implementation that follows the current rules (no React, controller + LitElement, light DOM, `tests/` layout).
- New files: `manifest.json`, `backend.ts`, `controller.ts`, `nuxy-tool-angrysearch.ts`, `frontend.ts`, `types.ts`, `settings.json`, `locales/{en,tr,ja}.json`, `tests/backend.test.ts`, `tests/controller.test.ts`, `tests/nuxy-tool-angrysearch.test.ts`.
- Preserved behavior: search-as-you-type with a 3-character minimum and 150ms debounce, LIKE vs. REGEXP (via a custom SQLite `REGEXP` function) query modes, configurable search result limit, periodic background re-indexing on a configurable interval, "open file" / "open containing folder" actions, and the ignored-roots safety list that prevents the indexer from descending into `/proc`, `/dev`, `/sys`, `/snap`, `/run`, `/tmp`, `/var/run`, `/var/lock` when scanning from `/`.
- Replaced the four legacy hooks (`useAngrysearchData`, `useAngrysearchActions`, `useAngrysearchKeyboard`, `useAngrysearchSync`) and the `AngrysearchItem.tsx` component with a single `AngrysearchController` class (state, IPC, keyboard actions via `window.core.shell.registerShellActions`) and a light-DOM `LitElement` (`nuxy-tool-angrysearch`) rendering a single-panel results list with `nuxy-list`/`nuxy-list-item`/`nuxy-empty-state`.
- Manifest permissions trimmed to the minimal real set actually used by the backend: `fs`, `db`, `shell` (the legacy manifest also declared `storage`, which the backend never called).
- Verified `DbHandle.function()` (used to register the custom `REGEXP` SQLite function) and FTS4 virtual tables are both supported by the current `core.db` runtime (`packages/extension-host/src/core-proxy.ts` opens a `node:sqlite` `DatabaseSync`, whose native `.function()` and `fts4` support were confirmed directly) — **no changes to `packages/core` or the runtime were needed**, since the type and implementation already existed.
- `pnpm -C src test`, `pnpm typecheck`, and `pnpm lint:fix` all pass with zero errors in the new files (44/44 new tests green; the only failing tests in the full suite are 3 pre-existing, unrelated failures in `download-manager`, `notes`, and `ui-default/Kbd`).
## `clipboard` extension — ported from legacy architecture

- Added `extensions/clipboard/` as a full rewrite of the legacy React-based clipboard manager extension, targeting the current LitElement + controller + `CoreContext` architecture (see `rules/EXTENSION_GUIDE.md`, `rules/LIT_MIGRATION_GUIDE.md`, `rules/FRONTEND_STRUCTURE_GUIDE.md`).
- `backend.ts` — unchanged behavior from the legacy backend: clipboard polling (text + image, configurable interval), history persisted via `core.storage`, pin/unpin, delete, clear, copy-to-clipboard (text/image), copy-file-to-clipboard with file-existence check, max-unpinned-items cap, `storeImages` setting toggle. Logic ported almost verbatim, since the `CoreContext` surface (`core.clipboard.*`, `core.storage.*`, `core.settings.*`, `core.fs.fileExists`) already matched the legacy SDK shape — no `packages/core` changes were needed.
- `controller.ts` — new `ClipboardController extends BaseExtensionController`, merging the legacy `useClipboardHistory` (polling refresh), `useClipboardActions` (copy/pin/unpin/delete with optimistic state), `useClipboardKeyboard` (now `registerShellActions`-based footer/menu actions, following the `notes`/`nyaa` pattern instead of the deprecated `useToolKeyActions` hook), `useOmniBarSync` (now `core.shell.controlOmniBar`), and `useSelectedItemMeta` (image dimension probe + file-existence probe) into one class.
- `nuxy-tool-clipboard.ts` — LitElement, light DOM, `<nuxy-two-panel>` list (left) + preview/properties (right), mirroring the `nyaa` extension's two-panel pattern. File-not-found alert renders via `<nuxy-alert variant="danger">` instead of a hand-rolled banner. Pin indicator, file/url/image leading icons, and color/image/text preview all preserved from the legacy `ClipboardLeftPanel`/`ClipboardRightPanel`/`ClipboardPreview`/`ClipboardItemLeading`/`FileIconFor` components, rebuilt as Lit templates using `window.UI` components (`nuxy-list`, `nuxy-list-item`, `nuxy-item-leading`, `nuxy-properties-panel`, `nuxy-icon`, `nuxy-alert`, `nuxy-empty-state`) instead of React/JSX.
- `utils/history.ts` and `utils/itemType.ts` ported essentially unchanged (pure logic, no `window.UI` dependency) — same for their test suites.
- `frontend.ts` reduced to a one-line bootstrap (`import './nuxy-tool-clipboard.ts'`).
- Settings (`pollIntervalMs`, `maxHistoryItems`, `storeImages`) and locales (`en`, `tr`, `ja`) ported from the legacy extension, with an added `search.placeholder` key for the new search-placeholder integration.
- No prior `clipboard` extension existed in this repo's git history — nothing to reconcile with `task/download-manager` or other prior work.
- Manifest: `com.nuxy.clipboard`, type `tool`, permissions `["clipboard", "storage", "fs"]` (matches actual `core.clipboard.*`, `core.storage.*`, `core.fs.fileExists` usage).
- Tests: `tests/history.test.ts`, `tests/itemType.test.ts`, `tests/backend.test.ts` (ported, all passing), plus new `tests/controller.test.ts` and `tests/nuxy-tool-clipboard.test.ts` for the new controller/element layer. Full `pnpm -C src test`, `pnpm typecheck`, `pnpm lint`, and `pnpm format` are green.

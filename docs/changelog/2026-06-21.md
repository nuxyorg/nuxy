# Changelog — 2026-06-21

## `ollama` extension — ported from legacy architecture

- Added `extensions/ollama/` — a full rewrite of the legacy React-based Ollama chat extension (`/ollama` in the old extension generation) onto the current LitElement + controller architecture.
- `backend.ts` — `register(core)` exposes `chat`, `query`, `models`, `health`, `configure`, `getConfig`, `history:save/load/clear`, plus `eval`/`openWithQuery` for omnibar provider integration. Registers both `registerTool` and `registerProvider`, matching the `calculator`/`notes` provider convention. Settings (`host`, `model`, `thinkingColor`, `systemPrompt`, `temperature`) are read/written via `core.settings`, never `core.storage`. `core.storage` is used only for `history.json`. `fetch` is used directly against `${host}/api/chat` and `${host}/api/tags` — confirmed available as a permission-gated global in `packages/extension-host/src/index.ts` (requires the `network` permission, already declared).
- `controller.ts` — `OllamaController extends BaseExtensionController<OllamaState>`, merging the legacy `useOllamaData`/`useOllamaActions`/`useOllamaKeyboard`/`useOllamaSync` hooks into one class: loads config/history/models on connect, sends chat via a streaming `fetch` to `/api/chat` (`stream: true`), supports message queueing while a response is in flight, and registers footer/menu key actions (`Enter` send/queue, `Escape` stop, clear-history and model-switch menu entries) via `window.core.shell.registerShellActions`.
- `nuxy-tool-ollama.ts` — LitElement, light DOM (`createRenderRoot() { return this }`), renders chat history with the existing `nuxy-chat-message` UI-kit component, `nuxy-empty-state` for the zero-message state, and `nuxy-alert` for errors/queued-message notices. No custom input — text arrives via the `query` property setter from the omnibar, matching the no-own-input rule.
- `frontend.ts` is a one-line bootstrap (`import './nuxy-tool-ollama.ts'`).
- Backend uses `core.i18n.t()` for all provider-facing strings instead of the legacy hardcoded `"Ask Ollama"` / `"ollama — \"...\""` literals; locales (`en`, `tr`, `ja`) cover the new `provider.*` keys plus the original UI strings.
- Simplified relative to the legacy version: the `thinkingColor` setting is preserved and still configurable, but the gradient/window-border "thinking" animation (`nuxy-gradient-toggle` custom event) was intentionally **not** wired up, since that event is not in the documented approved `window.dispatchEvent` channel list (`rules/EXTENSION_GUIDE.md` §5.11) and is only otherwise used internally by the `gradient` helper extension's legacy bridge.
- Tests: `tests/backend.test.ts` (23 cases, ported 1:1 from the legacy spec plus new system-prompt/temperature coverage), `tests/controller.test.ts` (streaming chat, HTTP failure, blank-input guard, queueing), `tests/nuxy-tool-ollama.test.ts` (manifest tag resolution, query/committedQuery/extensionId property forwarding, connect/disconnect lifecycle). All 32 new tests pass; `pnpm typecheck` and `pnpm exec eslint extensions/ollama` are clean.
